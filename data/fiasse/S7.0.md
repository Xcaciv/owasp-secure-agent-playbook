# S7 Roles and Responsibilities in SSEM Adoption

## S7.0 Security's Role

Security Assurance takes joint responsibility pertaining to the timely delivery of the end result. This ensures the focus of Software Engineers remains on building securable code through a self-governed process with verified outcomes.

### Responsibility Model

| Responsibility | Description |
| :--- | :--- |
| Guidance through participation | Participate in development activities like design and requirements gathering |
| Establish guardrails | Provide guardrails for development to operate as intended |
| Metrics interpretation | Security metrics measure effective partnership with development, not developer adherence |
| Value through enablement | Focus on requirements, design, and assurance — not policing or micromanaging |

### Key Distinctions

| Principle | Description |
| :--- | :--- |
| Joint responsibility | Security shares responsibility for timely delivery |
| Not a measure of adherence | Application security posture reflects how well expectations are set, practices applied, and items allowed to be worked |
| Quality foundation | Application security will never outperform quality assurance — quality is a foundation for securable software |
| Cultural influence | If posture stays stagnant, adjustments to development culture and/or leadership may be necessary |
| Scalable impact | Spending time in design activities allows AppSec to guide a larger number of developers |
