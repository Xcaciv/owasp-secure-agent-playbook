# S8 Evolution and Adoption of FIASSE

## S8.1 Adapting to Emerging Software Engineering Trends

FIASSE is positioned to evolve alongside emerging trends in software engineering.

### S8.1.1 AI-Driven Software Development

FIASSE provides a framework for **Prompt Engineering for Securable Code** — crafting prompts that request code exhibiting SSEM attributes:

| SSEM Attribute | Prompt Engineering Application |
| :--- | :--- |
| Analyzability | Direct AI to generate well-commented code with clear naming conventions, coding standards, and logical structure |
| Modifiability | Guide AI to produce modular code with low coupling, high cohesion, clear interfaces, no hardcoded config |
| Testability | Request inherently testable code with clear inputs/outputs, minimal side effects, co-generated unit tests |
| Reliability | Request code that implements sanitization, predictable execution, and defensive patterns |

Additional integration points:
- Include security requirements and SSEM attributes in Product Requirements Documents (prd.md)
- Include SSEM attributes in Acceptance Criteria Documents (acd.md)
- Include SSEM attributes in code standards for generation (e.g., copilot-instructions.md)

### S8.1.2 Low-Code and No-Code Platforms

### S8.1.3 Cloud-Native and Serverless Architectures

| Emphasis | Description |
| :--- | :--- |

### S8.1.4 Continuous Security Engineering

| Adaptation | Description |
| :--- | :--- |
| Avoid "Shovel Left" in automation | Enhanced guidance on avoiding anti-patterns in security automation |
| Observability | Improved actionable security intelligence through real-time monitoring |
| Developer-friendly processes | Continued development of processes that integrate with developer workflows |

### S8.1.5 AI and Security Governance

With AI playing a larger role in software development, security considerations and robust governance become critical. OWASP projects focused on AI security provide further guidance.
