# S5 Addressing Common AppSec Anti-Patterns

## S5.2 Strategic Use of Security Output

While scanning and testing tools are valuable for understanding security posture, their output must be used strategically.

### Key Principles

| Principle | Description |
| :--- | :--- |
| No implicit requirements | Security requirements should not be assumed implicit; developers should not be blamed for missing controls if expectations were not set |
| Respect workflows | Productive engineers have structured workflows; disruptions lead to degraded quality |
| No bypass | Fix requests should never circumvent established development processes |
| Clear actionability | AppSec should not expect developers to act on findings without clear, actionable information |
| Proper execution | Developers need the opportunity to properly execute their workflow when addressing security issues |

Bypassing established workflows leads to misunderstandings and mistakes — the very flaws that Application Security is trying to prevent.
