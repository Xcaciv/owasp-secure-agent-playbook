# S8 Evolution and Adoption of FIASSE

## S8.2 Organizational Adoption Strategies

Adopting FIASSE does not require a rigid structured approach, but organizations can benefit from strategic implementation.

### Recommended Steps

| Step | Description |
| :--- | :--- |
| 1. Assess Current Practices | Evaluate organizational openness; identify existing security practices and SSEM alignment; determine understanding of software engineering principles; assess misalignments FIASSE can address |
| 2. Integrate SSEM Terminology | Incorporate SSEM attributes (Maintainability, Trustworthiness, Reliability) into developer documentation, coding standards, style guides, and training materials to socialize concepts and provide common language |
| 3. Identify Key Influencers | Find senior software engineers and stakeholders who can internalize and champion FIASSE; they should have strong software engineering understanding |
| 4. Educate and Train Teams | Provide comprehensive, role-specific training integrated into onboarding and continuous learning; primary ongoing delivery through merge reviews, architecture discussions, and requirements gathering |
| 5. Foster Collaboration | Promote regular AppSec-Dev collaboration; encourage AppSec to engage in development activities like requirements gathering rather than reviewing in isolation |
| 6. Continuously Monitor | Implement real-time security observability; use insights to refine security strategies and FIASSE implementation |

### Critical Training Principle

FIASSE must be discussed **in the context of software engineering**, not as a separate security initiative. After initial training, primary delivery should be through:
- Merge reviews
- Architecture discussions
- Requirements gathering

### Applicable Organizations

| Organization Type | Application |
| :--- | :--- |
| Large technology companies | Integrating into Continuous Security Engineering |
| Financial institutions | Enhancing data protection through SSEM attributes |
| AI and cloud-based companies | Designing secure architectures using FIASSE principles |
| Open-source projects | Adopting secure development guidelines based on SSEM |
