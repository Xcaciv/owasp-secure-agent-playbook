# S3 The Securable Principle

## S3.4 Measuring SSEM Attributes

### S3.4.2 Measuring Trustworthiness

#### S3.4.2.1 Confidentiality

| Metric | Type | Description |
| :--- | :--- | :--- |
| Identified Data Leaks | Quantitative | From penetration tests, code reviews, or incidents |
| Access Control Violations | Quantitative | Logged unauthorized access attempts (prevented or successful) |
| Data Classification Adherence | Qualitative | Review of how well data is classified and protections align |
| Least Privilege Review | Qualitative | Assessment of whether components and users have only necessary permissions |
| Encryption Effectiveness | Qualitative | Review of algorithms used, key management practices |

#### S3.4.2.2 Accountability

| Metric | Type | Description |
| :--- | :--- | :--- |
| Audit Log Coverage | Quantitative | Percentage of critical system actions logged with sufficient detail |
| Traceability Success Rate | Quantitative | Percentage of audited actions uniquely traceable to an entity |
| Audit Log Review Findings | Qualitative | Results from periodic reviews for completeness and usefulness |
| Non-repudiation Strength | Qualitative | Strength of evidence linking actions to entities (e.g., digital signatures) |

#### S3.4.2.3 Authenticity

| Metric | Type | Description |
| :--- | :--- | :--- |
| Authentication Failures | Quantitative | Number of failed login attempts (may indicate brute-forcing) |
| Strong Authentication Use | Quantitative | Percentage of authentication points using MFA or strong credential types |
| Identity Verification | Qualitative | Review of processes for verifying user and system identities |
| Auth Mechanism Integrity | Qualitative | Assessment of security of authentication protocols and implementations |
