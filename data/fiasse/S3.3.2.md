# S3 The Securable Principle

## S3.3 Securable Code Strategy

### S3.3.2 SSEM as a Design Language

By defining attributes using established software engineering terminology, SSEM provides a **common design language**. This enables developers to discuss, reason about, and implement security considerations using concepts already familiar to them.

### Conversation Shift

| From (Traditional) | To (SSEM) |
| :--- | :--- |
| "Is it secure?" | "Do we meet our goals for this attribute?" |
| Myopic vulnerability treatment | Cohesive creation of inherent securable qualities |
| Find-and-fix monotony | Intentional design from the design phase |
| Security jargon isolation | Shared engineering terminology |

### Cultural Impact

A design language can bring together a culture of quality made up of diversely inclined professionals to focus on common goals. This cultural alignment can strongly influence the product's internal structure to support these technical values.

Security plays a crucial role by equipping teams to highlight key context-specific considerations, contrasting with a find-and-fix approach that fails to scale.
