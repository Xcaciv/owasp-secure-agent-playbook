# S6 Practical Guidance for Secure Software Development

## S6.5 Dependency Management

Dependency management starts before a library is introduced. The library should be evaluated for being fit to support the system and its values.

### SSEM-Based Dependency Evaluation

| SSEM Attribute | Evaluation Criteria |
| :--- | :--- |
| Analyzability | Understand full scope, transitive dependencies, specific purpose, potential attack surface; maintain clear inventory |
| Modifiability | Design loosely coupled dependencies for easier updates, patching, or replacement |
| Testability | Ensure dependencies can be managed during testing (mocking, stubbing, version pinning) |
| Trustworthiness | Verify source and integrity (signed packages, checksums, trusted repositories) |
| Reliability | Assess how dependency failure modes impact overall system reliability; plan mitigations |

### Key Practices

| Practice | Description |
| :--- | :--- |
| Minimize dependencies | Each dependency introduces additional maintenance needs |
| Regular updates | Integrate dependency updates into sprints or regular maintenance cycles |
| Mitigation for unfixable flaws | When no direct update fixes a known flaw, identify compensating controls, forking/patching, or re-implementation |
| Clear organizational policy | Define processes for use and maintenance of open-source dependencies |
| Beyond CVE scanning | Thoroughly analyze dependencies and their transitive "baggage"; CVE reliance alone is insufficient |

Once FIASSE has been adopted, potential dependencies can be evaluated against the values of the culture it creates, ensuring alignment with SSEM attributes.
