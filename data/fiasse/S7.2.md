# S7 Roles and Responsibilities in SSEM Adoption

## S7.2 Guiding Developing Software Engineers

Software engineers still gaining experience benefit greatly from mental models like SSEM and FIASSE methodologies. While they may struggle to intuitively know "What can go wrong?", they can learn through structured guidance, merge reviews, and pair programming.

FIASSE provides the context for **why** certain practices are important, helping build robust engineering intuition — a taste and smell for good and bad code.

### Developing Engineers Should Be Encouraged To

| Practice | Description |
| :--- | :--- |
| Focus on engineering fundamentals | Becoming strong software engineers first |
| Ask for clear requirements | Request Security Requirements and Acceptance Criteria |
| Exercise dependency caution | Avoid introducing new external dependencies without due diligence |
| Seek code reviews | Actively request reviews from senior team members to learn and build confidence |
| Write thorough tests | Cover exceptional conditions and out-of-bounds values |
| Learn defensive coding | Particularly input validation and output encoding |
| Understand trust boundaries | Be mindful of trust boundaries and seek to deeply understand the architecture |
| Study FIASSE and SSEM | Learn how fundamental principles affect outcomes |
