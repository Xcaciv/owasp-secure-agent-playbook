# S4 Integrating Security into Development Strategy

## S4.2 Natively Extending Development Processes

A key principle for reducing friction is to integrate security into existing development workflows rather than imposing separate, external security gates.

Security often takes the reviewer seat. While assurance is important, it is more impactful to be **involved in requirements, architecture, and design**. Security positioned as a partner is likely to be better informed and in a better position to provide value.

### Strategic Extensions

| Extension | Description |
| :--- | :--- |
| Architecture considerations | Consider business concerns to emphasize security implications (e.g., throttling and DDoS protection to preserve business integrity) |
| Predefined checklists | Develop flexible checklists incorporating SSEM attributes and security considerations for various contexts |
| Usability framing | Frame usability as building trust from a security standpoint (e.g., clear error messages, intuitive permissions management) |
