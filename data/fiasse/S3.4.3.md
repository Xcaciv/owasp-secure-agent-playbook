# S3 The Securable Principle

## S3.4 Measuring SSEM Attributes

### S3.4.3 Measuring Reliability

#### S3.4.3.1 Availability

| Metric | Type | Description |
| :--- | :--- | :--- |
| Uptime Percentage | Quantitative | e.g., 99.99% |
| Mean Time Between Failures (MTBF) | Quantitative | Average time between system failures |
| Mean Time To Recovery (MTTR) | Quantitative | Average time to restore service after failure |
| Redundancy Review | Qualitative | Assessment of system redundancy for critical components |
| Disaster Recovery Test Results | Qualitative | Validated outcomes from DR testing exercises |

#### S3.4.3.2 Integrity

| Metric | Type | Description |
| :--- | :--- | :--- |
| Data Corruption Incidents | Quantitative | Number of detected data corruption events |
| Checksum/Hash Validation Rate | Quantitative | Success rate for data at rest and in transit |
| Input Validation Effectiveness | Qualitative | Review of input validation mechanisms at trust boundaries |
| File Integrity Monitoring Alerts | Qualitative | System file integrity monitoring results |

#### S3.4.3.3 Resilience

| Metric | Type | Description |
| :--- | :--- | :--- |
| RTO Adherence | Quantitative | How often Recovery Time Objectives are met after incidents |
| Performance Under Stress | Quantitative | System metrics during load testing or simulated attacks (e.g., DDoS) |
| Defensive Coding Practices | Qualitative | Assessment of input validation, output encoding, robust error handling |
| Incident Response Effectiveness | Qualitative | Review of system and team recovery from security incidents or operational failures |
