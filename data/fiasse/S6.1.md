# S6 Practical Guidance for Secure Software Development

## S6.1 Establishing Clear Expectations

Clear expectations are foundational to building securable products. AppSec can maximize its impact by setting expectations effectively, which requires alignment with the business processes involved in software development.

### S6.1.1 Proactive Communication

| Practice | Description |
| :--- | :--- |
| Inform development teams | About new testing initiatives or security programs pertaining to their product |
| Demonstrate tools | To those interested, fostering collaboration and identifying key contacts |
| Regular sync points | Offer support and maintain momentum |
| Soft skills | Essential for AppSec professionals to be effective in this role |

### S6.1.2 Integrating Security into Requirements

Active AppSec participation in formal requirements gathering is crucial. This shifts security from a post-development review to an integral component of the product.

| Deliverable | Description |
| :--- | :--- |
| Security Features | Specific features to implement: authentication mechanisms, encryption requirements, access controls |
| Threat Scenarios | Potential misuse cases or attack paths relevant to the feature; helps identify necessary controls |
| Security Acceptance Criteria | Specific, testable conditions a feature must meet; enables QA to perform security testing by verifying requirements |

By embedding security into foundational design decisions via requirements, attributes like Trustworthiness, Integrity, and Fault Tolerance are more effectively realized as part of the standard workflow.
