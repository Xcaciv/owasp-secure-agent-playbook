# S2 Foundational Principles

## S2.5 Aligning Security with Development

True alignment between security and development requires a return to first principles. Instead of imposing security-centric jargon and processes that may slow or disrupt development, FIASSE advocates for using **well-established software engineering terms** to describe securable code attributes.

SSEM properties like Analyzability, Modifiability, Testability, Confidentiality, etc. foster understanding and empower developers to address security confidently without years of dedicated security experience.

### Alignment Benefits

| Benefit | Example |
| :--- | :--- |
| Developer confidence | Developers address security using concepts they already understand |
| Security recognition | Security identifies highly Analyzable code by clarity, low cyclomatic complexity |
| Proactive identification | Recognizing securable attributes helps identify security weaknesses before they are exploitable |
| Rapid response | Testable code can be altered quickly with high confidence in quality outcomes |
| Partnership | Development adds to Security's confidence by understanding what building securable software entails |

This also requires specific participation from AppSec professionals in the early stages of the SDLC, particularly during requirements gathering and feature planning.
