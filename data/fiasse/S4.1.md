# S4 Integrating Security into Development Strategy

## S4.1 Applying SSEM to Dependency Management

SSEM principles are not limited to first-party code; they are equally critical when managing software dependencies. Proactively applying SSEM attributes to the selection, integration, and maintenance of third-party libraries is essential for building securable systems.

| SSEM Attribute | Dependency Application |
| :--- | :--- |
| Analyzability | Understand each dependency's full scope, transitive dependencies, purpose, and attack surface; maintain a clear inventory |
| Modifiability | Design loosely coupled dependencies to facilitate updates, patching, or replacement |
| Testability | Ensure dependencies can be effectively managed during testing (mocking, stubbing, version pinning) |
| Trustworthiness | Verify source and integrity (signed packages, checksums, trusted repositories) |
| Reliability | Assess how dependency failure modes impact overall system reliability and plan mitigations |

Detailed practical guidance on dependency management through the SSEM lens is provided in S6.5.
