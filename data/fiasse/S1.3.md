# S1 Introduction

## S1.3 Document Purpose and Scope

The purpose of FIASSE is to introduce:
- **FIASSE** (/feiz/) — Framework for Integrating Application Security into Software Engineering: the overarching strategic approach and practices
- **SSEM** (/si:m/) — Securable Software Engineering Model: a common design language and set of principles within FIASSE

The term 'securable' emphasizes the dynamic nature of software security (see S2.1). Because of this dynamic nature, Application Security must align with business objectives for software development.

### Intended Audience

AppSec professionals, software engineers, engineering managers, and anyone involved in software development who seeks to improve application security outcomes.

### AppSec vs. Product Security

| Role | Focus |
| :--- | :--- |
| AppSec | Technical security aspects; works directly with developers to integrate SSEM attributes; responsible for security assurance |
| Product Security | Strategic level; translates business risk into security requirements; manages customer communications; ensures regulatory compliance |

FIASSE serves as a unifying framework for both roles: SSEM provides the technical language and measurable attributes, while the framework emphasizes business alignment and clear expectations.
